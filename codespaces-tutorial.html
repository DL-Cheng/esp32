📝 Codespaces 同步指南

如何在 Codespaces 中新增檔案並同步到 GitHub

📚 本指南將引導您完成整個流程： 從在 GitHub Codespaces 中建立或編輯檔案，到使用 Git 命令將變更同步推送到 GitHub 倉庫。 無論您是初學者還是有經驗的開發者，本指南都能幫助您快速完成這些操作。

📖 分步驟指南

1

開啟 GitHub Codespaces

在您的 GitHub 倉庫頁面上，點擊 <Code> 按鈕，然後選擇 Codespaces 標籤。

點擊 建立 Codespaces on main（或您的當前分支名稱）。

GitHub 倉庫首頁 ↓ 點擊 <Code> 按鈕 ↓ 選擇 Codespaces 標籤 ↓ 點擊 「建立 Codespaces on main」 ↓ 等待 VS Code 環境加載
💡 提示

第一次啟動可能需要 1-2 分鐘。Codespaces 提供了完整的開發環境，就像在本地執行 VS Code 一樣。

2

檢查您的倉庫文件

在 VS Code 左側邊欄可以看到「檔案總管」，其中列出了您倉庫中的所有檔案。

您可以瀏覽現有檔案的結構，熟悉您的專案佈局。

Codespaces ├── 檔案總管（左側邊欄） │ ├── index.html │ ├── README.md │ ├── .gitignore │ └── 其他檔案... └── 編輯器區域（中央）
3

建立新檔案（或編輯現有檔案）

建立新檔案：右鍵點擊檔案總管中的資料夾，選擇「新增檔案」，輸入檔案名稱。

編輯現有檔案：直接點擊檔案並在編輯器中修改內容。

示例：建立新的 HTML 文件
右鍵點擊根目錄 → 選擇 「New File」 → 輸入 tutorial.html → 在編輯器中輸入內容

✅ 當檔案有未保存的更改時，檔案名稱旁會出現白點「●」

4

保存檔案

在 Codespaces 中，按 Ctrl+S（Windows/Linux）或 Command+S（Mac）保存檔案。

或者使用菜單：File → Save

💡 提示

Codespaces 支援自動保存。您可以在設定中啟用此功能：File → Preferences → Settings，搜尋「Auto Save」並選擇您的偏好。

5

開啟終端

點擊上方菜單中的 Terminal，然後選擇 New Terminal，或按 Ctrl+`（反引號）。

終端會在 VS Code 下方打開。

VS Code 窗口 ├── 頂部菜單欄 │ └── Terminal → New Terminal └── 下方面板 └── 終端（命令行界面）
6

檢查 Git 狀態

在終端中輸入以下命令來檢查您的修改狀態：

查看 Git 狀態
git status

輸出將顯示：

On branch main Your branch is up to date with 'origin/main'. Changes not staged for commit: (use "git add <file>..." to stage) modified: codespaces-tutorial.html Untracked files: (use "git add <file>..." to include in what will be committed) new-file.html
7

新增檔案到 Git（Staging）

在終端中執行以下命令，將所有修改（包括新檔案）新增到 Git 的暫存區：

新增所有改動的檔案
git add .

或新增特定檔案：

新增特定檔案
git add codespaces-tutorial.html new-file.html
💡 說明

git add . 會新增當前目錄下所有未追蹤或已修改的檔案。. 代表「當前目錄」。

8

提交變更（Commit）

執行 commit 命令，為您的更改添加一條有意義的提交訊息：

提交更改
git commit -m "Add Codespaces tutorial page"

提交訊息格式說明：

  • -m：指定提交訊息的標誌
  • 訊息應簡潔明瞭，用英文或中文說明此次提交的目的
  • 良好的實踐是使用祈使句，例如：「Add」、「Fix」、「Update」

✅ 提交成功後，您會看到類似的輸出：
[main a1b2c3d] Add Codespaces tutorial page

9

推送到 GitHub（Push）

執行 push 命令，將本地提交推送到遠端 GitHub 倉庫：

推送到 GitHub
git push

系統會自動推送到您的預設遠端分支（通常是 origin main）。

💡 說明

如果您正在使用具體的分支，也可以執行：git push origin branch-name

✅ 推送成功後，您會看到類似的輸出：
To github.com:yourname/repo.git
[main a1b2c3d..e4f5g6h] main -> main

10

驗證 GitHub 上的變更

回到 GitHub 網站，刷新您的倉庫頁面，確認新檔案已出現。

您可以看到：

  • 新增的檔案在檔案列表中
  • 提交歷史中出現您的提交訊息
  • 提交者和時間戳記

⚡ 完整 Git 工作流程

1️⃣ 一次性設定（首次使用時）
# 設定 Git 使用者名稱和郵箱 git config --global user.name "Your Name" git config --global user.email "your.email@example.com" # 查看設定 git config --list
2️⃣ 標準工作流程（每次提交）
# 步驟 1：查看修改狀態 git status # 步驟 2：新增所有修改 git add . # 步驟 3：提交修改（包含有意義的訊息） git commit -m "Description of your changes" # 步驟 4：推送到 GitHub git push
3️⃣ 其他常用命令
# 查看提交歷史 git log --oneline # 查看特定檔案的修改 git diff filename.txt # 撤銷上一次提交（謹慎使用） git reset --soft HEAD~1 # 克隆遠端倉庫到本地 git clone https://github.com/username/repo.git # 拉取最新更改 git pull

❓ 常見問題

Q: 我忘記新增檔案就直接提交了，怎麼辦？
A: 您可以執行 git add . 再次新增檔案，然後執行 git commit --amend 覆蓋上一次提交。
git add . git commit --amend --no-edit
然後重新推送：git push --force-with-lease
Q: 提交訊息有誤，如何修改？
A: 執行以下命令修改最後一次提交的訊息：
git commit --amend -m "Corrected commit message"
然後重新推送。
Q: 推送時出現「Permission denied」錯誤，怎麼辦？
A: 這通常是因為 SSH 金鑰或認證令牌設定有誤。確保：
  • 您的 SSH 金鑰已正確添加到 GitHub 帳戶
  • 或使用個人存取令牌（Personal Access Token）進行認證
  • Codespaces 環境中的 Git 認證已正確設定
Q: 我想在推送前查看我的修改，如何操作？
A: 使用以下命令查看差異：
git diff # 查看所有未暫存的修改 git diff --staged # 查看已暫存但未提交的修改
Q: 我意外刪除了一個檔案，如何恢復？
A: 如果檔案未被提交，可以執行：
git checkout -- filename.txt
如果檔案已被提交，可以查看歷史版本。

🎯 Git 提交最佳實踐

✓ 好的提交訊息範例

git commit -m "Add user authentication feature" git commit -m "Fix bug in email validation" git commit -m "Update documentation for setup process" git commit -m "Refactor database connection logic"

✗ 應避免的提交訊息

git commit -m "asdf" git commit -m "update" git commit -m "fixed bug" git commit -m "..."

📌 提交訊息的關鍵點：

  • 清晰描述所做的變更
  • 使用英文或中文，保持一致
  • 使用祈使句（"Add"、"Fix"、"Update" 等）
  • 第一個單詞首字母大寫
  • 簡潔且有意義（通常 50 字以內）

🚀 後續步驟

現在您已掌握基本的 Codespaces 和 Git 工作流程。下一步您可以：

  • 學習分支管理（git branch、git checkout）
  • 學習合併分支（git merge）
  • 學習拉取請求（Pull Request）工作流程
  • 設定 GitHub Actions 進行自動化測試
  • 在不同的 Codespaces 環境中進行協作開發
